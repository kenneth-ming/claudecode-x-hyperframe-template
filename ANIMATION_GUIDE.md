# 動畫生成規則(給 Claude Code 讀)

> 當使用者提供 SRT 台詞時,請**嚴格依照這份規則**,在 `index.html` 生成對應的動畫。
> 目標:不同的台詞,都產出**同一套視覺風格**的動畫(配色、版面、敘事手法一致)。

---

## 一、開始前的工作流程(務必照這個順序)

1. 先讀使用者的 SRT,理解每一句在講什麼、整段的故事走向。
2. 規劃每句台詞對應的 icon / 文字 / 動作 / 時間點。
3. **先用白話跟使用者說明你的動畫規劃(每句配什麼),等使用者確認後再開始寫。**
4. **分批增量寫入** `index.html`:先寫骨架存檔,再每次加 1~2 個元素,**絕對不要一次輸出整份**(會超過輸出上限)。
5. 寫完跑 `npx hyperframes lint` 跟 `npx hyperframes validate`,再請使用者 `preview`。

---

## 二、基本設定

- **比例**:依使用者指定。IG 直式用 `9:16 / 1080x1920`;YT 橫式用 `16:9 / 1920x1080`。
- **影片**:`assets/my-footage.mp4`。
  - 若做「滿版影片 + 玻璃疊層」:影片用 `object-fit: cover` 鋪滿整個畫面當底層,保留音訊,長度對齊影片。
  - 若做「透明疊層(回剪映合)」:不放影片,`html / body / 容器` 全部透明背景。
- **長度**:composition 長度對齊台詞片段的總長度。

---

## 三、品牌規範(務必遵守,不可改)

- **顏色**:深藍 `#0A2540`、金 `#C9A227`、灰藍 `#8895A7`、白 `#F5F7FA`、金色光暈 `#E6C55C`。
- **絕對不要**用螢光藍、霓虹色,或範例殘留的其他品牌色。
- **中文字體**:Noto Sans TC(用 `assets/fonts` 或 Google Fonts 載入)。中文字多時載入完整字體,不要用 text 子集。
- **icon**:一律用 **Lucide 線條風格**(inline SVG),線條金色或藍色,粗細約 2.5,大小約 110~120px。
- **玻璃質感**(若用玻璃模板):玻璃卡半透明、能透出後面的影片,保留折射光感。

---

## 四、時間軸換算

- SRT 的秒數若**不是從 0 開始**(例如從 02:39 起),請把所有秒數**減掉第一句的起始秒數**,換成「從 0 起算」的相對時間。
- 每一句台詞對應一個動畫動作,動作出現在那句話的時間點。
- 最後一句結束後,留約 0.5~1 秒做收尾(淡出或停在高潮畫面)。

---

## 五、敘事風格(這是靈魂,最重要)

1. **用「icon 連貫說故事」,不要做成一頁一頁切換的資訊圖。**
   - 要有一個**主角**(通常是 AI bot 或一個角色)貫穿全場,會移動、會跟場景互動。
   - 元素用「推進 / 累積 / 轉場」銜接,不要每段把舊的清掉再換新的。
   - 用「位置移動」說故事(例如角色從左走到右),不要原地跳字。

2. **用顏色說故事。**
   - 痛點 / 負面 / 還沒解決 → 用**灰色調**(灰藍、去飽和)。
   - AI 介入 / 正面 / 解決後 → **轉成金色**,氛圍變亮。
   - 「灰 → 金」的轉變要明顯,這通常是一段動畫的轉折點。

3. **最後一句通常是高潮。** 用**金色大字 + 金色光暈 + 打勾 / 標籤**強調(例如「值錢」「準時送達」「成交」),要做得有份量。

4. **泡泡 / 卡片 / 資料行**用「由下往上滑入 + 淡入」,**一個一個累積出現**,不要一次全冒出。

---

## 六、版面規則

- **透明疊層(疊在口播影片上)**:元素放畫面**右側或上半部**,左側 / 下方留空給**人臉與字幕**,絕對不要擋臉。下方約 28% 留給字幕。
- **滿版影片 + 玻璃疊層**:玻璃卡浮在畫面**上半部**(垂直約 12%~45%),不要蓋到中下方的臉。
- 文字夠大、留白夠多,確保手機上看得清楚。

---

## 七、常見動畫類型(對照 SRT 內容挑一種)

- **資料卡逐行型**:台詞在「列舉項目」(像 A、像 B、像 C)→ 一張卡片,資料一行一行浮現,最後框起來強調。
- **流程旅程型**:台詞在描述「步驟 / 過程」(先…然後…接著…)→ 一個角色沿路徑推進,從起點走到終點。
- **對話泡泡型**:台詞是「對話 / 問答」→ 做成聊天介面,泡泡一個個滑入累積。
- **掃描整理型**:台詞在講「整理資料 / 被讀懂」→ 散亂(灰)→ 對齊(金)的轉變。
- **數字反差型**:台詞有「強烈數字對比」(10萬 vs 50塊)→ 大數字被劃掉、跳出反差數字。
- **灰金對比型**:台詞在講「A 不行、但 B 可以」→ 左灰右金的對比。

---

## 八、技術紀律(不可省略)

- **分批增量寫入**,元素多時尤其重要,避免超過單次輸出上限。
- 寫完一定先 `lint` + `validate` 再 `preview`。
- **輸出方式**:
  - 透明疊層 → `npx hyperframes render --format mov`(ProRes 4444,回剪映用)。
  - 含影片成品 → `npx hyperframes render`(輸出 mp4)。
  - **不要用 preview 畫面裡的 Export 按鈕**(會失敗),一律用 CLI render。
- 真實品牌名稱(如瑞幸、星巴克)**只用文字呈現,不要放它們的 logo**(商標問題)。

## 九、自我檢查(寫完問自己)

- [ ] 配色是不是只用了藍金那幾個色?有沒有殘留別的品牌色?
- [ ] 中文有沒有正確套上 Noto Sans TC?
- [ ] 是「連貫說故事」還是變成「一頁頁切換」?(要前者)
- [ ] 有沒有擋到人臉 / 字幕區?
- [ ] 最後的高潮夠不夠有份量?
- [ ] 是不是分批寫入、有沒有 lint + validate?
